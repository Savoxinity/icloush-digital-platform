# iCloush智慧工厂——Phase030-Sprint3中文展柜与支付预研-V3.0.0

## 版本概述

本次 Phase030 对应 Sprint 3 的继续开发，重点不再是泛化的 B2B 展示，而是将 web-b2b 前台进一步收束为 **中文主场的 2C 高端零售数字展柜**。页面层面完成了 Showroom 与 PDP 的中文文案重写、思源字体分层、实验数据面板与新的 fallback 转化弹窗；支付层面则预埋了统一的 Payment Gateway Interface，并补充了微信支付 V3 / 支付宝验签说明文档，为后续真实密钥到位后的快速接入建立边界。

## 本次更新内容

| 模块 | 更新内容 | 结果 |
| --- | --- | --- |
| web-b2b Showroom | 将主标题、导语、系列信息与转化入口改为中文为主，保留必要英文术语 | 前台语义更贴近高端零售与顾问式转化 |
| web-b2b 视觉系统 | 引入思源黑体、思源宋体与 IBM Plex Mono，强化标题、正文、参数三级字体分层 | 中文阅读质感与科幻感同步提升 |
| Showroom / PDP 深化 | 加强错落瀑布流、深色留白、实验数据面板与硬边按钮 | 6:3:1 方尖碑体系在前台落得更完整 |
| fallback 通道 | 将申请配额弹窗升级为“空间节点交易通道合规接入中”版本，加入企业微信与小程序占位 | 在真实支付未开通前保持转化承接 |
| admin → 前台联通 | 复核商品池 CRUD、主图上传与 specs 动态录入对前台展示链路的支撑 | 后台录入商品仍可被前台读取与渲染 |
| payments 基建 | 预埋 `PaymentGatewayInterface`、`createPaymentOrder`、`paymentWebhookCallback` | 为微信/支付宝真实接入预留统一接口 |
| 文档 | 新增支付网关资料摘录与接入说明文档 | 后续可直接据此推进证书、验签与回调实现 |

## 验证结果

本次改动后，`apps/web-b2b` 的类型检查已通过；前台相关 Vitest 断言已更新并通过；针对 admin 到前台的商品联通验证也已重新执行通过。由此可确认，当前 Sprint 3 的中文化前台与支付预研工作已经回到可交付状态。

## 交付备注

本次版本仍保留“申请配额”作为前台主转化入口，尚未直接对接真实微信支付或支付宝收银链路。后续当 ICP、商户号、证书、公钥与 API 密钥正式下发后，可优先从服务端 `createPaymentOrder` 与 `paymentWebhookCallback` 的正式实现继续推进，而不需要再重构当前前台页面结构。

# 环洗朵官网前端 VI 升级方案 V2

本轮重构将环洗朵从“常规 B2B 企业官网”切换为更接近**工业科技发布终端**的视觉范式。整体目标不是营造温和亲切的销售页，而是通过**大屏画面、硬边界栅格、极少量高光色与更强的信息纪律**，建立一种更高价值、更可信、同时更接近消费电子旗舰站点的品牌认知。

| 模块 | V1 旧问题 | V2 设计原则 | 当前已落地方式 |
| --- | --- | --- | --- |
| 首页 Hero | 信息先于画面，视觉震慑不足，仍偏常规卡片官网 | 先画面、后参数、再动作 | 改为 100vh 级 Hero，左侧为标题与信任信号，右侧为大屏媒体承载结构 |
| 顶部导航 | 圆角按钮、页面感较重，缺少“终端”感 | 导航做薄、边界做硬、动作做少 | 使用 1px 栅格导航、仅保留核心栏目和主动作 |
| 技术模块 | 解释性太强，缺少参数系统感 | 每个技术点都像可接入真实检测字段的参数板 | 改为三列硬栅格模块，预留真实参数和报告接入位 |
| 解决方案 | 三张常规卡片，缺少行业部署感 | 更像任务频道与场景分发器 | 改为三段式场景调度行，强调动作入口 |
| 产品矩阵 | 普通产品卡片，缺少规格与工业蓝图感 | 产品展示必须同时包含对象、规格、用途与动作 | 改为双栏工业蓝图式矩阵，统一承载 TDS 与试样入口 |
| 申样转化 | 表单虽清楚，但视觉等级不够 | 保持高压画面下的直接转化锚点 | 改为硬边界表单区与渠道区，强化客服、小程序与技术直连路径 |

## 视觉语言

| 维度 | 规则 |
| --- | --- |
| 主背景 | 使用 `#F3F6F8`、`#EEF3F7`、`#FFFFFF` 三个浅色层级构建工业级洁净感 |
| 强调色 | 仅保留 `#002FA7` 作为主高光色，用于导航焦点、主 CTA、技术标签和边界线 |
| 文字色 | 主文字接近黑色，辅助文字使用黑色透明度降低，而不再依赖彩色文字强调 |
| 边框系统 | 统一 1px 直角边界与 gap-px 栅格，避免圆角卡片的幼态感 |
| 排版 | 标题使用高密度粗黑字重，参数与标签使用等宽、超大字距的 mono 风格 |
| 画面策略 | 优先大图、大视频、大留白；当正式媒体资源未接入时，用工业抽象构图维持版面张力 |

## 后续执行建议

第一，尽快把真实的 4K Hero 大图或短片接入当前媒体承载位，替换占位构图。

第二，为每个产品补齐真实 TDS 下载文件、应用参数、适用行业与包装规格，以便把矩阵页自然延展到单品 PDP。

第三，将同一套直角栅格、浅底大屏和蓝色高光规则继续推送到 `/care` 与后续小程序承接页，避免生态内视觉层级断裂。

# 环洗朵参考站审计摘要（2026-04-21）

本轮参考重点对比了 DJI 首页与 Nothing 商城列表，目标是为环洗朵科技官网提炼一种更有压迫感、但仍保持专业可信的工业终端式界面语言。

| 参考站 | 观察结论 | 可迁移到环洗朵的实现方式 |
| --- | --- | --- |
| DJI 官网 | 首页不是传统网页，而是连续的满屏产品画面。每一屏都由单一主产品、超少量文案和极克制按钮构成，用户先被画面抓住，再逐步理解产品。导航极薄、层级非常轻，真正的视觉主角永远是大图与巨物感。 | `/tech` 首页应改为 100vh 大图/大视频驱动，减少首屏文字，突出一个核心标题、一句技术副标和 1–2 个主动作。内容模块切换成接近分屏硬切的节奏，而非传统 B2B 官网的碎片滚动。 |
| DJI 官网 | 大疆虽然信息量大，但并不靠密集卡片堆砌，而是用“产品英雄画面 + 简单 Learn More / Buy”建立消费电子级的信任与转化路径。 | 环洗朵的“申请试样 / 下载 TDS / 查看方案”也应被压缩成少量高价值动作，并绑定单一产品或单一解决方案画面，而不是平均分散在整屏按钮和说明文里。 |
| Nothing 商城 | 商品陈列采用高度克制的白底、黑字、极细分隔和网格化列表。产品本身成为版面秩序的一部分，参数和价格呈现出一种说明书式、装置式的冷静感。 | 环洗朵产品矩阵应改造成参数化信息板与工业目录，不再使用传统圆角卡片。可用 1px 边框、表格式分区、等宽字体参数和硬朗直角栅格强化“工业终端”气质。 |
| Nothing 商城 | 分类筛选清晰但不花哨，视觉重心始终停留在产品和网格系统，而非装饰性的色彩与背景。 | 环洗朵的解决方案导航和产品筛选应采用硬边界目录结构，弱化装饰，避免渐变、玻璃、圆润按钮等稚嫩语汇。 |

综合来看，环洗朵当前页面最大问题不是“信息不够多”，而是**画面不够强、秩序不够硬、产品不像高价值技术对象**。新版方向应从“传统企业官网”切换为“工业科技品牌发布终端”：以满屏大图、极简导航、硬栅格参数板、少量高光色和极强版面纪律建立专业性与未来感。

## 开发预览验收补记

在当前 `/tech` 开发预览中，新版首屏已经明显脱离原先的普通企业官网卡片感，具体表现为：第一，Hero 采用了接近满屏的工业媒体承载区，左侧标题、右侧大屏视觉、底部参数板的关系已经接近 DJI 式“先画面、后参数”的节奏；第二，导航、场景分发、产品矩阵和申样入口全部切换到直角栅格和硬边界体系，视觉纪律明显增强；第三，蓝色高光被压缩到少数关键 CTA 与技术标签，避免了过度装饰。

当前仍保留一项刻意的占位策略：Hero 区域中的 `4K HERO / USER MEDIA READY` 说明，表示页面已经为正式的高清图像或短片预留好了结构，但还需要用户后续提供真实媒体资源完成最终质感封顶。

## 内联资源回退验收

由于当前开发预览中的 `/manus-storage` 代理对新旧资源均返回异常，本轮为保证预览阶段的真实可见性，已将环洗朵首屏 Hero 图与占位版 TDS 下载先切换为内联资源方案。浏览器侧验证结果显示：Hero 图像已成功加载，实际尺寸为 `845 × 475`；TDS 入口已输出 `data:application/pdf;base64,` 前缀并保留新窗口打开行为，因此新版 `/tech` 在预览中已经具备可视化 Hero 与可点击下载动作。

## 第二轮视觉整改验收补充

在最新 `/tech` 预览中，环洗朵首页已切换为深色工业终端式首屏：背景不再使用 Base64 占位图，而是通过 `/manus-storage/huanxiduo-hero-4k_a47d4dd9.jpg` 加载真实高清 Hero 图片，并叠加代码驱动的动态流体场与噪点扫描纹理。浏览器侧进一步确认了主滚动容器为 `main.h-screen.snap-y.snap-mandatory.overflow-y-auto`，其 `scrollSnapType` 为 `y mandatory`，且 5 个核心 section 均具有 `scrollSnapAlign: start` 与整屏高度，说明强制吸附滚动底座已生效。

当前版本的动态首屏采用“高清静态图 + Canvas 代码动态场 + 噪点遮罩”的组合方案，已满足在未接入正式短片资源时的高压动态替代要求。后续如果补入实验室视频素材，可继续在同一 Hero 容器内无缝切换为真实 video background，而无需推翻现有分屏节奏与参数化布局。

# iCloush LAB. H5 首版设计说明

## 项目背景

本轮页面改版以用户提供的《iCloush LAB. 品牌感官系统手册 V4.1》摘要要求、Atmospheric Purification 系列视觉素材以及用户在对话中追加的界面约束为依据，目标不是做一张常规电商首图，而是把 **“视觉暴力”** 作为筛选机制：先用压迫性的形式语言完成用户分层，再把产品转化为一种“战术物资”与“空间重组协议”的叙事对象。

这种方法对应的不是温和导购，而是 **Brutalist Deco、Space Jeweler、Archivist Tone、Practical Sci-Fi** 的复合表达。页面因此主动降低生活方式电商的亲和感，转而强化冷峻秩序、材料张力、战术术语与高密度留白，让用户在第一屏就感知到该系列的价格带、专业壁垒与稀缺性。

## 提炼出的核心原则

| 视觉原则 | 手册/需求中的原始意图 | 页面中的具体落地 |
| --- | --- | --- |
| 压迫式巨型排版 | 用户要求首屏采用顶格巨型字、夸张留白与强张力阅读节奏 | Hero 使用 `ATMOSPHERIC / PURIFICATION` 双行超大标题，行距压缩至 0.9，并将文字直接压在深黑背景与光晕图像之上 |
| 黑域主导的色彩秩序 | 用户明确要求全局背景为 `#050505`，主色为黑、枪灰、温暖米白，只保留极少信号色 | 全局主题切换为 dark，`index.css` 中将背景、卡片、边框与前景变量统一重构为黑域体系，页面中仅保留米白文本与弱金属边线 |
| 档案员而非导购员语气 | 用户要求文案像“密封档案被调取”，避免普通电商语感 | Hero 说明、状态面板、分区标题和 CTA 全部改为协议式命名，例如 `ARCHIVE STATUS`、`TACTICAL PAYLOAD`、`INITIATE PROTOCOL`、`ORBITAL ASSETS` |
| 微距材质对撞 | 用户要求“粗糙矿物 vs 精密蓝瓶”的高反差物料组合 | 首屏使用 `视觉锤.webp` 作为背景锤击图；后续分区接入蓝瓶、部署矩阵、嗅觉档案与包裹礼盒素材，形成粗野与精密的连续对照 |
| 机械切割感而非柔性圆角 | 用户强调需要切削、斜切、工业珠宝感 | 通过 `clip-panel` 与 `clip-cta` 统一实现斜角切边、机械卡片和按钮外轮廓，替代常规平滑卡片 |
| 品牌入口先完成筛选再谈转化 | 用户要求首屏必须同时保留 AP/FC 导航与登录入口，但不能做成常规商城 | 顶部导航仅保留 `AP / FC / ORBITAL ASSETS` 与 `[ REQUEST ACCESS ]`，主 CTA 则使用 `[ INITIATE PROTOCOL ]` 与 `[ ORBITAL ASSETS ]`，兼顾筛选和转化 |
| 视觉系统先为 AP 系列服务，再为 FC 预留扩展 | 用户当前明确以 Atmospheric Purification 为第一优先级，Fabric Care Deluxe 为第二阶段 | 页面结构以 Hero、AP Payload、Olfactory Archive 为主轴，在底部单独保留 `MATERIAL_STASIS` 分区作为 FC 的冷启动骨架 |

## 页面结构映射

页面结构遵循“**先震慑、再验证、后预埋扩展**”的顺序。首屏只解决三个问题：第一，告诉用户这不是普通家清产品；第二，给出 AP 系列的协议化身份；第三，把登录与咨询入口埋在强视觉秩序中，而不是用常规商城按钮破坏氛围。为此，首屏采用了左文案、右状态面板的不对称布局，让大字与查询结果并存，既有品牌气氛，也保留统一平台真实摘要数据。

第二层结构是 `TACTICAL PAYLOAD / AP-01`。这一层不继续解释品牌故事，而是把 AP-01 定义成空间重组装置，并通过产品图、礼盒图和部署原则卡片，完成“为什么贵、为什么危险、为什么值得继续下滑”的解释。第三层 `OLFACTORY ARCHIVE` 则承接嗅觉档案与部署矩阵，将原本容易被做成香氛生活方式内容的模块，转换成更接近实验室档案与武备目录的版面。

最后一层是 `MATERIAL_STASIS`。这里没有抢跑 FC 正式内容，而是故意把它处理成低温静置状态，以免稀释 AP 的首屏主导权。这样做可以保证首版页面既具备向下扩展的结构，又不会在视觉中心上失焦。

## 关键交互与转化逻辑

导航中的 `AP / FC / ORBITAL ASSETS` 采用锚点跳转，目的是让用户能在单页结构中快速切换系列与资产说明，而不必进入多层路由。`[ REQUEST ACCESS ]` 保留登录导向，服务于用户要求的“login 入口”；`[ INITIATE PROTOCOL ]` 保留合作/咨询入口，服务于“转化入口”；`[ ORBITAL ASSETS ]` 则作为更偏产品资产与采购入口的辅助按钮。这样，页面没有把品牌 H5 和商城强行拆开，而是通过不同语气的 CTA 完成双重目标。

当前预览中，LAB 联系配置状态显示为 `fallback`。结合服务端数据结构可知，LAB 的 fallback 配置本身就定义了 `/account` 与 `/shop` 两个目标地址，因此在未录入更完整数据库配置前，页面仍能保持可用的跳转逻辑。这意味着首版可以先稳定承接入口，再决定是否要把 `business` 场景配置进一步补齐到数据库层。

## 当前结论

首版已经把用户要求中的 **高张力 Hero、冷峻导航、AP 先行、FC 预埋、登录入口与转化入口并存** 全部落在了可运行页面上。它仍然只是第一版，因此未来仍可继续把非线性滚动阻尼、更多微距材质镜头、以及 FC 正式内容继续深化；但就当前版本而言，视觉方向、结构方法与前端落地之间已经形成了可核验的一致关系。

# 支付网关官方资料摘录

微信支付官方文档《总述-APIv3如何签名和验签》说明，在 **请求应答场景、接口回调场景、调起支付场景** 中，开发者都需要进行签名验签。该页同时明确，请求微信支付接口时商户需要使用 **商户 API 证书** 生成请求签名，而返回应答与回调通知则需要使用 **微信支付公钥或平台证书** 完成验签；导航中还单列了“如何解密微信支付回调报文”，说明正式回调链路不仅要验签，还要处理报文解密。[1]

支付宝开放平台《自行实现验签》说明，如果未使用 SDK，则需要自行实现验签流程。官方将其区分为 **同步验签** 与 **异步通知验签** 两种模式；同步返回时，开发者只对支付宝返回 JSON 中 `xxx_response` 的值做验签，而异步通知则需要在去除 `sign`、`sign_type` 后，对剩余参数进行 `url_decode`、字典排序、拼接字符串，再以 RSA 验签。文档同时指出，开放平台已推出符合 OAS3.0 规范的 V3 OpenAPI，并推荐无法使用 SDK 的开发者直接对接。[2]

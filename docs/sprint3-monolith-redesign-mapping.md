## 6:3:1 黄金美学如何落到代码

### 60%：巨物粗野（Monolithic Mass）

“巨物粗野”在代码中的实现，不是通过体积阴影、玻璃高光或饱和渐变制造“高级感”，而是通过**超大字号、低密度内容区、巨型器物容器与大面积纯黑留白**制造压迫感。`ShowroomPage` 首屏以 `RETAIL MONOLITH.` 作为唯一超大标题，使用 `font-[Syncopate]`、极低行高和 `max-w-[8ch]` 限制，形成类似方尖碑铭文的竖向压迫；核心对象则由 `ProductArtifact` 输出的高比例容器承载，并通过大面积空场与稀疏线条让用户先感知“体量”，再读取产品。[1]

| 设计意图 | 代码落点 | 说明 |
| --- | --- | --- |
| 巨型标题压迫 | `App.tsx` 第 356-362 行 | 首屏标题 `RETAIL MONOLITH.` 使用极大字号、紧凑行高和极窄字块宽度 [1] |
| 巨物对象承载 | `App.tsx` 第 260-300 行 | `ProductArtifact` 使用大面积黑色对象容器、中心重物布局与边界线构成“黑碑”感 [1] |
| 负空间与单件陈列 | `App.tsx` 第 433-480 行 | `full / wide / stacked` 版式与不对称栅格替代传统货架网格 [1] |

### 30%：绝对材质（Ascetic Material）

“绝对材质”的关键是**边界、切角、线性结构与纯色本体**。当前实现中没有依赖圆角、玻璃模糊或柔和阴影，而是使用 `clip-path`、hairline 网格、单层边框和纯黑底板形成生硬的物理材料感。`monolith-panel` 与 `monolith-button` 采用统一的倒角轮廓，产品图层使用多个 1px 线条与极暗底色叠出“裸露材质”而非装饰性包裹。[1][2]

| 设计意图 | 代码落点 | 说明 |
| --- | --- | --- |
| 45° 倒角容器 | `App.tsx` 第 261、269、441、616 行；`index.css` 第 74-87 行 | 主体容器与按钮通过 `clip-path: polygon(...)` 实现切角 [1][2] |
| Hairline 边框/网格 | `App.tsx` 第 262、269-272、441-442 行；`index.css` 第 58-64 行 | 采用 1px 网格与极暗边界，不用投影制造层次 [1][2] |
| 纯黑材质基底 | `App.tsx` 第 333、440、520 行；`index.css` 第 4-28 行 | 以 `#000000 / #020202 / #050505` 作为核心材质，而非玻璃或渐变面板 [1][2] |

### 10%：神性纹章（Divine Vector）

“神性纹章”不是大面积装饰，而是**极少量、极薄、极克制**的仪器感符号：准星、信号线、坐标化参数、等宽微字与小体量徽记。当前实现中，`Crosshair`、`micro-copy`、`micro-pill`、品牌徽记 `monolith-badge` 及信号色只在边角和数据标签处出现，形成审判式、仪器式的冷感，而不是赛博朋克式霓虹堆叠。[1][2]

| --- | --- | --- |
| 极简准星 | `App.tsx` 第 252-253、267-268、561 行；`index.css` 第 117-144 行 | 以 1px 十字准星替代发光图标 [1][2] |
| 微型等宽参数 | `App.tsx` 第 264-265、356、388、423、556、619 行；`index.css` 第 66-72、102-115 行 | 使用等宽小字表达参数与档案感 [1][2] |
| 神性高光压缩 | `App.tsx` 第 41-44、293-299、393-395、534 行 | 高光仅保留在 AP/FC 信号线、标签边框与少量文本强调处，避免成为主视觉主体 [1] |

## 去 Apple / Glassmorphism 的重构方式

本轮活跃零售页面没有采用 iOS 式圆角卡片、毛玻璃面板、液态浮层或以阴影制造亲和层次。相反，页面在结构上统一采用**直角基础 + 倒角切面 + 纯黑底 + 实线边框 + 硬切反白按钮**。`monolith-button` 的 hover/focus 样式是直接反白，不使用渐变过渡；对话层与对象卡片均以纯色与边框构成，不依赖 `backdrop-blur` 或阴影。[1][2]

| 反潮流法则 | 当前实现 |
| --- | --- |
| 不使用圆润药丸/液体玻璃 | 活跃零售页的主体容器采用 `clip-path` 倒角与直角区块，而非圆角卡片 [1][2] |
| 不使用模糊玻璃 | `index.css` 未定义任何 retail 专属 `backdrop-blur` 工具类；零售主界面结构由纯黑底板、实线边框和网格组成 [2] |
| 不使用柔性阴影 | 当前活跃零售页的主体结构不依赖 `shadow-*`，层次由边界线与稀疏网格建立 [1][2] |
| 按钮反馈必须硬切 | `monolith-button` hover/focus 直接切换为白底黑字，不使用玻璃或渐变 [2] |

> 说明：本结论当前仅针对 `App.tsx` 挂载的活跃零售展示路径成立。仓库中仍存在历史模板组件与示例页，后续补证任务会继续通过检索与测试明确哪些属于非活跃遗留代码、哪些需要继续清理。

## 2C 零售语义与命名修正

当前前台对外叙事已从偏 B2B 的 “showroom / product / platform” 迁移至更贴合高端零售展示的 **`retail / gallery / object`**。查询层调用 `trpc.retail.galleryObjects` 与 `trpc.retail.objectDetail`，首屏文案改写为 `RETAIL MONOLITH.`、`PRIVATE ALLOCATION GATE.` 与 `INSPECT OBJECT`，强调的是顾问式零售转化，而不是企业团购申请。[1]

| 原方向 | 当前方向 | 证据 |
| --- | --- | --- |
| showroom / product / platform | gallery / object / retail | `App.tsx` 第 638-665、703-711 行 [1] |
| 企业采购语义 | 高端零售与私域顾问式转化语义 | `App.tsx` 第 361、494-496、599-608 行 [1] |

## 当前仍需继续补证的边界

尽管活跃零售路由已经切换为方尖碑语言，但为了在下一个检查点前形成**更严格的工程证据**，仍需继续推进以下工作：一是补充自动化断言，验证活跃展示页中确实存在倒角容器、准星、hairline 网格、等宽参数与巨幅标题；二是对 `apps/web-b2b/src` 中的历史模板组件进行分层检索，明确哪些属于不再挂载的旧资产、哪些需要继续清理，以避免把“活跃路径完成重构”误写成“整个前台源码完全净化”。

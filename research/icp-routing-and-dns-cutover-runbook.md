# iCloush 备案路径与域名切流执行手册

作者：**Manus AI**  
日期：2026-04-12

## 一、文档目的

本手册用于解决 iCloush 当前最关键的两个执行问题。第一，已经存在的小程序相关备案记录，究竟应当如何与官网、商城、后台及 API 的大陆部署衔接。第二，在腾讯云中国内地轻量应用服务器已确定的前提下，正式域名应当按什么顺序完成备案、解析与上线切换。

腾讯云官方文档明确区分了**首次备案、接入备案、新增服务（原备案在腾讯云 / 原备案不在腾讯云）、变更备案**等多种路径，并指出使用中国内地地域轻量应用服务器开展网站业务时必须先完成备案。[1] 同时，备案域名需要满足实名认证、注册有效期、注册商合规以及**备案主体信息与域名实名信息一致**等条件。[1] [2]

## 二、先判断“已有备案号”属于哪一类资产

当前您提到的备案信息是与“小程序 / 智慧工厂 ERP 测试”相关的记录，因此它不能自动等同于“iCloush 官网已经可以直接上线”。备案判断的核心，不是“有没有备案号”这么简单，而是要看该备案号背后对应的是**哪个主体、哪个接入商、哪类互联网服务以及是否已覆盖当前准备上线的站点结构**。

| 核查项 | 要确认的内容 | 影响的判断 |
| --- | --- | --- |
| 主体名称 | 是否为“上海环洗朵科技有限公司” | 决定能否沿用主体备案 |
| 备案号 | 是否为主体级有效备案记录 | 决定是否需要首次备案 |
| 当前接入商 | 是腾讯云还是其他云厂商 | 决定走接入备案还是腾讯云内新增服务 |
| 当前备案服务类型 | 小程序、APP、网站或其他 | 决定官网是否需要新增网站服务 |
| 当前备案网站信息 | 是否已存在与 `icloush.com` 对应的网站项 | 决定能否直接复用 |
| 域名实名主体 | 是否与备案主体完全一致 | 决定能否顺利提交域名备案 |

在实践中，只要上述六项中有一项不清楚，就不建议贸然提交备案。最稳妥的方式是先把它们核成一张表，再根据结果进入对应的备案路径。

## 三、备案路径判定矩阵

腾讯云官方规则可以被整理为下面这张执行矩阵。您只需要对照现实情况打勾，就能知道自己下一步该走哪条路径。[1]

| 现实情况 | 推荐备案动作 | 是否需要新的主体备案号 | 备注 |
| --- | --- | --- | --- |
| 公司主体从未做过任何备案 | **首次备案** | 是 | 从零开始提交主体与网站信息 |
| 公司主体已有备案，但接入商不在腾讯云 | **接入备案** 或 **新增服务（原备案不在腾讯云）** | 否 | 取决于现有服务覆盖情况 |
| 公司主体已在腾讯云有备案，且现在要新增 `icloush.com` 网站服务 | **新增服务（原备案在腾讯云）** | 否 | 最省事的理想路径 |
| 公司主体已备案，但企业名称 / 负责人 / 证件等信息发生变化 | **变更备案** | 否 | 先改主体，再上新站 |
| 域名实名主体与备案主体不一致 | 不能直接提审 | 不适用 | 先改实名再备案 |

如果您当前已确认 `icloush.com` 完成过户，那么下一步应优先核实：**过户后的实名主体是否已同步为备案主体公司全称**。如果这一点没有完全一致，即使云资源与域名都在腾讯云，备案仍有较高概率卡在前置核验环节。[1] [2]

## 四、当前项目最可能的三种落地场景

结合现有信息，iCloush 项目大概率会落在以下三种场景之一。为了便于您快速执行，我将它们转写成业务判断语言，而不是单纯照搬备案术语。

| 场景 | 现实描述 | 建议动作 |
| --- | --- | --- |
| 场景 A | 公司此前只有小程序或其他服务备案，没有官网网站项 | 在腾讯云提交**新增网站服务**相关备案流程 |
| 场景 B | 公司的网站备案在其他服务商，现准备把官网部署到腾讯云大陆服务器 | 在腾讯云做**接入备案** |
| 场景 C | 公司已经在腾讯云完成过网站备案，只是现在新增 `icloush.com` 或新网站服务 | 走**新增服务（原备案在腾讯云）** |

如果您暂时无法立刻确认备案资产归属，那么最务实的动作不是继续猜，而是先登录工信部备案查询系统、腾讯云备案控制台或现有服务商备案面板，把主体备案状态截图保存下来。这样后续无论是内部执行还是我继续为您整理材料，都会快很多。

## 五、备案申请顺序表

一旦备案路径确认，建议按固定顺序推进，不要交叉操作。因为备案最怕的是一边改实名、一边改解析、一边又换接入商，最后材料状态互相冲突。

| 顺序 | 动作 | 说明 |
| --- | --- | --- |
| 1 | 核验主体信息 | 确认营业执照、法人/负责人、联系方式一致 |
| 2 | 核验域名实名 | 确认 `icloush.com` 的实名主体与备案主体一致 |
| 3 | 核验腾讯云资源 | 确认中国内地轻量应用服务器已购买且时长满足备案要求 |
| 4 | 判断备案路径 | 在首次备案、接入备案、新增服务、变更备案中选定其一 |
| 5 | 整理网站服务信息 | 准备网站名称、服务内容、负责人信息、前置审批说明（如无则按常规企业网站申报） |
| 6 | 在腾讯云备案系统提交 | 按既定路径填写主体信息与网站信息 |
| 7 | 完成短信核验 / 人脸核验等 | 按腾讯云与管局要求完成后续验证 |
| 8 | 等待腾讯云初审与管局审核 | 审核期间不做破坏性变更 |
| 9 | 备案通过后再做正式解析切换 | 避免未备案先开站 |

腾讯云官方资料指出，轻量应用服务器用于备案时需为中国内地地域、包年包月、购买时长达到 3 个月及以上，并在备案期间保有足够剩余时长。[1] [3] 这意味着，如果您已经按截图中的轻量方案下单，那么资源方向基本是对的，重点只剩下“主体—域名—备案路径”的三者统一。

## 六、域名解析与正式上线的推荐切换顺序

备案与部署应分层推进。服务器可以买，环境可以先装，程序可以先部署验证，但**正式主域名的生产级解析**最好放在备案明确通过之后再执行。这是当前阶段最重要的风险控制动作之一。

| 阶段 | 推荐做法 | 原因 |
| --- | --- | --- |
| 服务器初始化 | 使用公网 IP、临时域名或测试子域调试环境 | 不影响正式域名合规状态 |
| 备案提交前 | 不全面开放 `icloush.com` 正式生产访问 | 降低未备案访问风险 |
| 备案审核中 | 完成 Nginx、Node、SSL 预配置、日志监控、防火墙与备份 | 为切流做准备 |
| 备案通过后第 1 步 | 解析 `icloush.com` 与 `www.icloush.com` 到生产入口 | 先让品牌主站稳定上线 |
| 备案通过后第 2 步 | 再逐步解析 `shop`、`account`、`api`、`admin` 等子域 | 分批放开，便于排错 |
| 切流后观察期 | 连续观察 3 至 7 天日志、错误率、登录与表单提交 | 避免全站同时暴露问题 |

## 七、建议的 DNS 记录落地方式

为了减少后期维护复杂度，建议所有对外站点优先使用统一的母域子域体系，而不是第一阶段同时让多个独立顶级域名各自承载一套站点。

| 记录 | 类型 | 指向建议 | 用途 |
| --- | --- | --- | --- |
| `@`（`icloush.com`） | A / CNAME | 指向生产接入层 | 主站入口 |
| `www` | CNAME | 指向主站或 301 到主站 | 统一入口 |
| `shop` | A / CNAME | 指向商城服务 | 商城前台 |
| `account` | A / CNAME | 指向客户中心服务 | 登录与订单查询 |
| `api` | A / CNAME | 指向 API 网关 | 后端接口 |
| `admin` | A / CNAME | 指向后台服务 | 管理后台 |
| `lab` | A / CNAME | 指向品牌站服务 | LAB 品牌站 |
| `tech` | A / CNAME | 指向品牌站服务 | 环洗朵科技站 |
| `care` | A / CNAME | 指向品牌站服务 | Care 品牌站 |

如果后续要启用 `icloushlab.com`、`icloushlab.cn` 或 `huanxiduo.cn`，更适合在**备案和主站上线稳定后**，作为第二阶段进行品牌跳转或独立站扩展，而不是现在同步摊开。

## 八、当前最应该立即执行的动作

基于您已经完成域名过户并确认轻量服务器规格的最新状态，我认为当前优先级最高的动作如下。

| 优先级 | 动作 | 结果 |
| --- | --- | --- |
| P0 | 核验 `icloush.com` 最新实名主体是否与备案主体完全一致 | 决定能否顺利提审 |
| P0 | 查询现有备案号的接入商与服务类型 | 决定走首次、接入还是新增服务 |
| P0 | 下单腾讯云中国内地轻量服务器并确认购买时长 | 锁定备案云资源 |
| P1 | 完成服务器安全初始化与部署环境准备 | 为审核期间的应用部署做准备 |
| P1 | 以 `icloush.com` 为主站准备网站名称、服务内容与备案材料 | 锁定主备案域 |
| P2 | 待备案通过后再逐步开放主域和各业务子域 | 控制上线风险 |

## 九、结论

就当前阶段而言，项目已经具备继续推进的两个关键条件：**主域名已经完成过户**，**大陆生产服务器方案已经基本锁定**。剩下真正决定进度的，不再是买不买服务器，而是尽快确认：现有备案号是否可作为腾讯云体系下的网站备案基础，以及 `icloush.com` 的实名主体是否已经与备案主体彻底对齐。

一旦这两个问题确认，iCloush 的官网、商城、客户中心、后台与 API 站点就可以按照“**先备案主域、后切主站、再放开子域**”的顺序稳妥上线。
